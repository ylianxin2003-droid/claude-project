# GitHub + Streamlit Cloud 部署指南

本文件夹可直接作为 **Git 仓库根目录** 推送到 GitHub，并在 Streamlit Cloud 上免费发布。

---

## 一、文件夹结构

| 文件 | 作用 |
|------|------|
| `app.py` | 网站入口（Streamlit 主程序，三种模式） |
| `config.py` | 配置 & 阈值（读取 `.env` 或云端 Secrets） |
| `serene_client.py` | SERENE API 客户端（官方 `Token` + `/api/calc/`） |
| `data_loader.py` | 数据加载 + 本地 fallback |
| `grid_generator.py` | 固定网格生成（6 个区域） |
| `map_builder.py` | 固定地图构建（grid + API + cache） |
| `map_cache.py` | 磁盘缓存层（Parquet / CSV） |
| `hazard_detector.py` | 危险检测引擎（空间梯度 + 时间变化率） |
| `alert_engine.py` | ICAO-style prototype advisories 生成 |
| `historical_runner.py` | 历史时间窗口分析 |
| `visualisation.py` | 所有 Plotly 图表 |
| `requirements.txt` | Python 依赖列表 |
| `data/*.json` | 本地样例数据（访客默认可用） |
| `.streamlit/config.toml` | Streamlit 配置 |
| `.streamlit/secrets.toml.example` | 云端密钥模板（仅供参考） |

**不要上传：** `.env`、`.streamlit/secrets.toml`、`data/cache/`、`*.parquet`

## 二、上传到 GitHub

### 方法 A：网页上传

1. GitHub 新建仓库（Public）
2. **Upload files**，把本文件夹内所有文件拖进去（保持目录结构）
3. Commit

### 方法 B：命令行

```powershell
cd "你的路径\AA-github-second"
git init
git add .
git commit -m "Aviation space weather dashboard v2"
git branch -M main
git remote add origin https://github.com/你的用户名/仓库名.git
git push -u origin main
```

---

## 三、Streamlit Cloud 部署

1. 打开 https://share.streamlit.io ，用 GitHub 登录
2. **Create app** → 选择刚建的仓库
3. **Main file path** 填：`app.py`
4. **Branch**：`main`
5. 展开 **Advanced settings** → **Secrets**，粘贴：

```toml
SERENE_API_BASE_URL = "https://spaceweather.bham.ac.uk"
SERENE_API_TOKEN = "你的token"
SERENE_API_TIMEOUT = "30"
SERENE_AUTH_SCHEME = "Token"
```

6. 点击 **Deploy**
7. 等待 1–3 分钟，获得链接：`https://<app-name>.streamlit.app`

## 四、三种模式说明

| 模式 | 用途 | 数据来源 |
|------|------|----------|
| **Existing dashboard** | 原有功能：多变量展示 + ICAO-style advisories | API / Local fallback |
| **Live fixed map** | 单时间点固定网格 + 危险检测 | SERENE API (point-based) |
| **Historical analysis** | 时间窗口历史分析 | API / Cache / Local data |

---

## 五、使用与安全建议

1. 公开网站建议访客默认用 **Local sample file** 或 **cached maps**，避免消耗 API 配额
2. **不要**把 token 写进代码或提交到 GitHub
3. 告警仅为 **prototype advisories**，非官方 ICAO 预警
4. `data/cache/` 目录自动生成，已在 `.gitignore` 中排除
5. Global 2.5° 网格需要约 10,500 次 API 调用，建议使用小区域或缓存数据

## 六、常见问题

| 问题 | 处理 |
|------|------|
| 部署失败、找不到模块 | 确认 `requirements.txt` 在仓库根目录 |
| API 不工作 | 检查 Streamlit Cloud → Settings → Secrets |
| 页面一直 loading | API 区域过大；用小范围，或选缓存数据 |
| 更新代码后网站没变 | Streamlit Cloud 一般会自动重建；在控制台点 **Reboot app** |
| Historical analysis 无数据 | API 当前只返回实时数据；使用缓存或 local fallback |

curl -X POST \

  -d latitude=52.4862 -d longitude=1.8904 \
  https://spaceweather.bham.ac.uk/api/calc/
